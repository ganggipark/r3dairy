# 콘텐츠 리뷰 체크리스트

> **용도**: 도메인 전문가가 생성된 일간 콘텐츠를 검수할 때 사용하는 체크리스트 템플릿
>
> **검수 대상**: `review-samples/YYYY-MM-DD.md` 파일 1개 = 1장의 체크리스트
>
> **작성 방법**: 각 항목을 확인한 후 `[ ]`를 `[x]`로 변경하고, 이슈 발견 시 하단 **이슈 기록** 섹션에 작성

## 1. 용어 준수 (Terminology Compliance)

> 사용자 노출 텍스트에서 내부 전문 용어가 노출되지 않는지 확인

### 1-1. 금지 용어 미노출 확인

- [ ] `사주명리`, `기문둔갑`, `사주` 등 동양 점술 용어가 사용자 노출 텍스트에 없음
- [ ] `천간`, `지지`, `간지`, `오행`, `십성`, `격국`, `용신`, `기신` 등 전문 한자 용어가 없음
- [ ] `세운`, `대운`, `신살`, `천을귀인`, `역마`, `공망`, `양인살` 등 전문 용어가 없음
- [ ] `NLP`, `알고리즘`, `엔진`, `계산`, `분석 모듈` 등 기술 용어가 없음
- [ ] `비견`, `겁재`, `식신`, `상관`, `정재`, `편재`, `정관`, `편관`, `정인`, `편인` 등 십성 용어가 없음

### 1-2. 허용 표현 사용 확인

- [ ] `오늘의 흐름`, `리듬`, `에너지`, `방향` 등 일반 언어 사용
- [ ] `집중 시간`, `주의 시간`, `좋은 방향` 등 친숙한 표현 사용
- [ ] 전문 개념을 일상 언어로 적절히 치환

## 2. 조사 정확성 (Korean Particle Accuracy)

> 역할 번역 시 단어 치환 후 한국어 조사(은/는/이/가/을/를/으로/로/과/와)가 올바른지 확인

### 2-1. 기본 조사

- [ ] `은/는` — 받침 유무에 따라 올바르게 사용됨 (예: "학습은" O, "업무는" O)
- [ ] `이/가` — 받침 유무에 따라 올바르게 사용됨 (예: "과제가" O, "결과물이" O)
- [ ] `을/를` — 받침 유무에 따라 올바르게 사용됨 (예: "업무를" O, "성과를" O)
- [ ] `으로/로` — 받침 유무에 따라 올바르게 사용됨 (예: "결과물로" O, "성적으로" O)
- [ ] `과/와` — 받침 유무에 따라 올바르게 사용됨 (예: "학습과" O, "의뢰와" O)

### 2-2. 역할별 치환 후 조사 확인

- [ ] **학생** 번역: 단어 치환(업무→학습, 프로젝트→과제 등) 후 조사 자연스러움
- [ ] **직장인** 번역: 단어 치환(학습→업무, 발표→보고 등) 후 조사 자연스러움
- [ ] **프리랜서** 번역: 단어 치환(업무→작업, 결정→계약 등) 후 조사 자연스러움

### 2-3. 문장 흐름

- [ ] 치환된 단어가 문장 맥락에서 어색하지 않음
- [ ] 이중 조사, 중복 조사 등 문법 오류 없음

## 3. 변형 다양성 (Role Variation Diversity)

> 3개 역할(학생/직장인/프리랜서) 간 의미 있는 차이가 있는지 확인

### 3-1. 역할별 차별성

- [ ] **학생** vs **직장인**: 핵심 키워드(공부↔업무, 과제↔프로젝트 등)가 다름
- [ ] **직장인** vs **프리랜서**: 관계 표현(동료↔클라이언트, 상사↔발주처 등)이 다름
- [ ] **학생** vs **프리랜서**: 결과물 표현(과제물↔납품물, 성적↔수익 등)이 다름

### 3-2. 의미 불변성 (Semantic Preservation)

- [ ] 3개 역할 모두 동일한 날의 동일한 리듬 정보를 담고 있음
- [ ] 긍정/부정 방향(추천/주의)이 역할 번역 후에도 유지됨
- [ ] 집중 포인트 개수가 역할 간 동일함
- [ ] 주의 포인트 개수가 역할 간 동일함
- [ ] 키워드 개수가 역할 간 동일함

### 3-3. 번역 과도함/부족함

- [ ] 번역이 과도하여 원문 의미가 왜곡된 경우 없음
- [ ] 번역이 너무 적어 역할 차별성이 없는 경우 없음

## 4. 최소 글자 수 (Minimum Content Length)

> 좌측 페이지(안내 콘텐츠) 핵심 블록 합산 400자 이상 확인

### 4-1. 전체 길이

- [ ] **학생** 번역: 핵심 블록 합산 **400자 이상** (리뷰 파일 하단 "콘텐츠 길이" 참조)
- [ ] **직장인** 번역: 핵심 블록 합산 **400자 이상**
- [ ] **프리랜서** 번역: 핵심 블록 합산 **400자 이상**
- [ ] **원본** 콘텐츠: 핵심 블록 합산 **400자 이상** (목표: 700~1200자)

### 4-2. 블록별 충실도

- [ ] 요약(Summary): 한 문장 이상, 맥락을 충분히 전달
- [ ] 리듬 해설: 2문장 이상, 오늘의 흐름을 구체적으로 설명
- [ ] 의미 전환: 구체적인 재해석 문장 포함 (요약 카드 단순 반복 아님)
- [ ] 리듬 질문: 사용자가 오늘 스스로 점검할 수 있는 구체적 질문
- [ ] 상태 트리거: 실천 가능한 제스처/문구/방법이 모두 채워짐

### 4-3. 금지 패턴

- [ ] "요약 카드만 나열" 패턴 없음 — 설명형 문단 반드시 포함
- [ ] 빈 항목(`(없음)` 표시) 3개 이하

### 메모
```
(실제 글자 수 기록 예: 학생 523자 / 직장인 548자 / 프리랜서 512자 / 원본 761자)
```

## 5. 톤 일관성 (Tone Consistency)

> 전체 콘텐츠에서 어조가 일관되고 사용자 친화적인지 확인

### 5-1. 어조 기준

- [ ] 친근하고 편안한 어조 유지 (딱딱한 공문서 어조 없음)
- [ ] 지나치게 단정적이거나 운명론적인 표현 없음 ("반드시", "절대" 등의 강압적 표현 최소화)
- [ ] 부정적 표현도 건설적으로 서술됨 ("이럴 수 있어요" 수준, 공포 조장 없음)

### 5-2. 역할별 톤 적합성

- [ ] **학생**: 응원하고 격려하는 어조 (또래 멘토 느낌)
- [ ] **직장인**: 전문적이고 실용적인 어조 (업무 효율 중심)
- [ ] **프리랜서**: 자율성과 주도성을 강조하는 어조 (자기결정 중심)

### 5-3. 7일 연속 일관성 (가능한 경우)

- [ ] 동일 역할로 7일 연속 읽었을 때 어조 급변 없음
- [ ] 매일 다른 키워드/표현이 사용되어 반복 지루함 없음
- [ ] 에너지 높은 날과 낮은 날의 톤 차이가 자연스럽게 반영됨
