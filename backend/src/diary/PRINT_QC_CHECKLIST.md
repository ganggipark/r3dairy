# 인쇄 전 검수 체크리스트 (Print Quality Control Checklist)

## 📋 체크리스트 개요
- **목적**: 주문형 다이어리 인쇄 전 최종 품질 검수
- **대상**: 운영자, 인쇄 담당자
- **소요시간**: 주문 1건당 약 5-10분

---

## 1. 주문 정보 확인 ✍️
### 고객 정보
- [ ] 고객명이 주문서와 일치하는가?
- [ ] 생년월일이 정확히 입력되었는가? (년/월/일)
- [ ] 출생시간이 있는 경우 정확히 반영되었는가?
- [ ] 성별(남/여)이 주문서와 일치하는가?

### 주문 사양
- [ ] 시작 날짜가 주문서와 일치하는가? (YYYY-MM-DD)
- [ ] 기간이 정확한가? (1개월/3개월/6개월/1년)
- [ ] 모드가 정확한가? (standard/large)
- [ ] 달력 타입이 명시되어 있는가? (양력/음력)

---

## 2. 표지 확인 📖
### 타이틀
- [ ] 메인 타이틀이 올바르게 표시되는가?
  - 기본: "라이프 리듬 다이어리"
  - 커스텀: 주문서의 productTitle과 일치
- [ ] 부제목이 표시되는가? "사주·기문둔갑 기반 개인화 일일 관리 다이어리"
- [ ] 컨셉 문구가 있는가? "나만의 리듬을 찾아 성장하는 시간"

### 소유자 정보
- [ ] 소유자 섹션에 이름이 표시되는가?
  - ownerLabel이 있으면 해당 값
  - 없으면 customerName 표시
- [ ] 소유자명이 깨지지 않고 선명한가?
- [ ] 소유자명 위치가 중앙 정렬되어 있는가?

### 기간 정보
- [ ] 사용 기간이 정확히 표시되는가? (시작일 ~ 종료일)
- [ ] 총 일수가 맞는가?
  - 1개월: 28~31일
  - 3개월: 89~92일
  - 6개월: 181~184일
  - 1년: 365~366일
- [ ] 생성일이 오늘 날짜로 표시되는가?

### 표지 디자인
- [ ] 표지 테두리가 온전한가? (3px 파란색 테두리)
- [ ] R³ System 브랜드 표시가 하단에 있는가?
- [ ] 전체적인 레이아웃이 균형잡혀 있는가?

---

## 3. 내부 페이지 확인 📄

### 페이지 수
- [ ] 전체 페이지 수가 예상과 일치하는가?
  - 표지 1페이지
  - 인덱스 페이지 (2~3페이지)
  - 일간 페이지 (총 일수만큼)
  - 주간 요약/반성 페이지 (주 수만큼)
- [ ] 페이지 번호가 순서대로 표시되는가?
- [ ] 누락된 날짜가 없는가?

### 일간 페이지 레이아웃
- [ ] 좌측 패널(오늘의 안내)이 표시되는가?
- [ ] 우측 패널(기록 공간)이 표시되는가?
- [ ] 하단 패널(사용자 기록)이 표시되는가?
- [ ] 날짜와 요일이 정확히 표시되는가?
- [ ] 음력 날짜가 표시되는가? (있는 경우)

### 콘텐츠 품질
- [ ] 한글이 깨지지 않고 정상 표시되는가?
- [ ] 특수문자(★, ●, □ 등)가 정상 표시되는가?
- [ ] 시간 슬롯이 순서대로 표시되는가? (09-11시, 11-13시...)
- [ ] 체크박스가 정사각형으로 표시되는가?

---

## 4. 모드별 검증 🔍

### Standard 모드 (기본)
- [ ] 본문 글꼴 크기: 10pt
- [ ] 페이지 여백: 10mm
- [ ] 줄 간격: 1.3배
- [ ] 제목 글꼴: 14pt
- [ ] 체크박스 크기: 3mm

### Large 모드 (큰 글씨)
- [ ] 본문 글꼴 크기: 12pt ⭐
- [ ] 페이지 여백: 15mm ⭐
- [ ] 줄 간격: 1.5배 ⭐
- [ ] 제목 글꼴: 18pt ⭐
- [ ] 체크박스 크기: 5mm ⭐
- [ ] 표지 타이틀: 42pt (standard는 36pt)

---

## 5. 파일명 확인 💾

### 명명 규칙
- [ ] 파일명 형식이 맞는가? `YYYYMMDD_고객명_기간_모드.pdf`
- [ ] 날짜가 시작날짜와 일치하는가?
- [ ] 고객명이 안전한 문자로 변환되었는가?
  - 공백 → 언더스코어(_)
  - 특수문자 → 언더스코어(_)
  - 한글 유지 (최대 20자)
- [ ] 기간 표시가 맞는가? (1m/3m/6m/1y)
- [ ] 모드 표시가 맞는가? (standard/large)

### 파일 위치
- [ ] `./order_output/` 디렉토리에 있는가?
- [ ] HTML 파일도 함께 생성되었는가? (선택)
- [ ] 파일 크기가 정상 범위인가?
  - 1개월: 500KB ~ 2MB
  - 3개월: 1MB ~ 5MB
  - 1년: 3MB ~ 15MB

## 6. 인쇄 전 최종 체크 ✅

### 인쇄 품질
- [ ] PDF가 정상적으로 열리는가?
- [ ] 모든 페이지가 A4 사이즈인가?
- [ ] 인쇄 미리보기에서 잘림이 없는가?
- [ ] 여백이 충분한가? (최소 5mm 이상)
- [ ] 양면 인쇄 시 홀수/짝수 페이지 정렬이 맞는가?

### 가독성
- [ ] 60cm 거리에서 본문이 읽히는가?
- [ ] 색상 대비가 충분한가? (특히 회색 텍스트)
- [ ] 표와 박스 경계선이 선명한가?
- [ ] 중요 정보가 강조되어 있는가?

### 제본 고려사항
- [ ] 제본 시 가려질 부분에 중요 정보가 없는가?
- [ ] 페이지 순서가 제본에 적합한가?
- [ ] 표지가 첫 페이지인가?

## 7. 재출력 판단 기준 🔄

### 즉시 재출력 필요 (치명적 오류)
- ❌ 고객명이 틀린 경우
- ❌ 생년월일이 틀린 경우
- ❌ 기간이 틀린 경우 (예: 1개월 주문인데 3개월 생성)
- ❌ 날짜가 누락된 경우
- ❌ 페이지가 누락된 경우
- ❌ 한글이 깨져서 읽을 수 없는 경우

### 고객 확인 후 재출력 (중간 오류)
- ⚠️ 소유자명이 다른 경우 (ownerLabel vs customerName)
- ⚠️ 출생시간이 누락된 경우
- ⚠️ 모드가 다른 경우 (standard vs large)
- ⚠️ 표지 타이틀이 다른 경우

### 재출력 불필요 (경미한 문제)
- ✓ 파일명이 규칙과 약간 다른 경우
- ✓ 생성일이 하루 차이나는 경우
- ✓ 페이지 번호 표시 위치가 약간 다른 경우
- ✓ 여백이 1-2mm 차이나는 경우

---

## 8. Large 모드 권장 상황 👴👵

### 반드시 Large 모드 권장
- [ ] 고객 연령 60세 이상
- [ ] 시력 관련 요청이 있는 경우
- [ ] "큰 글씨" 요청이 있는 경우
- [ ] 노안으로 가독성 문제를 언급한 경우

### Large 모드 고려
- [ ] 고객 연령 50-59세
- [ ] 선물용으로 연령대가 높은 수령인
- [ ] 필기 공간을 넓게 원하는 경우
- [ ] 프리미엄 버전 요청 시

### Large 모드 특징 안내
- 본문 글씨 20% 크게 (10pt → 12pt)
- 여백 50% 넓게 (10mm → 15mm)
- 체크박스 66% 크게 (3mm → 5mm)
- 전체 페이지 수 약간 증가
- 인쇄 비용 약간 증가 가능

## 9. 흔한 오류 사례 TOP 5 ⚡

### 1. 날짜 형식 오류
**증상**: 시작 날짜가 잘못 표시됨
**원인**: YYYY-MM-DD 형식이 아닌 다른 형식 입력
**해결**: 주문서의 startDate 형식 확인 후 재생성

### 2. 한글 이름 깨짐
**증상**: 고객명이 ??? 또는 이상한 문자로 표시
**원인**: 인코딩 문제 또는 특수문자 포함
**해결**: UTF-8 인코딩 확인, 특수문자 제거 후 재생성

### 3. 월말 날짜 처리 오류
**증상**: 1월 31일 + 1개월 = 2월 31일 (존재하지 않음)
**원인**: 월말 처리 로직 오류
**해결**: 종료일이 해당 월의 마지막 날로 조정되었는지 확인

### 4. 모드 지정 누락
**증상**: Large 모드 주문인데 Standard로 생성됨
**원인**: renderMode 파라미터 누락 또는 오타
**해결**: 주문서의 renderMode 확인 후 재생성

### 5. 파일명 특수문자 오류
**증상**: 파일이 생성되지 않거나 열리지 않음
**원인**: 고객명에 /, \, :, * 등 파일명 금지 문자 포함
**해결**: 고객명 정제 후 재생성
