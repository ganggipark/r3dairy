# ROLE_TRANSLATION - Role-based Content Variation Rules

## 1. Core Principle
- Rhythm(핵심 의미)는 유지
- 표현/예시/행동 가이드/질문만 역할에 맞게 변형

## 2. Roles
### Student
- Focus: 집중, 학습 리듬, 이해/암기 균형, 페이스 관리
- Avoid: 계약/보고/리더십/조직 정치 표현
- Replace terms:
  - "업무" → "공부/학습"
  - "회의" → "수업/과제/팀플"
- Daily question examples:
  - "오늘 가장 점수를 올릴 1가지 학습 행동은?"

### Worker
- Focus: 업무, 결정 타이밍, 커뮤니케이션, 관계/보고
- Avoid: 시험/과제/공부량 목표 표현
- Replace terms:
  - "공부" → "역량/학습(자기계발) 또는 업무 정리"
- Daily question examples:
  - "오늘 내가 통제 가능한 업무 결과는 무엇인가?"

### Freelancer / Self-employed
- Focus: 결정/계약/영업, 창작, 체력, 실행 리듬
- Avoid: 조직 고정 일정(회의/보고 강제) 전제 표현
- Daily question examples:
  - "오늘 나의 수익/성과에 직결되는 행동 1가지는?"

### General (default)
- Focus: 균형, 건강, 관계, 정리, 실행
- Daily question examples:
  - "오늘 가장 가벼운 한 걸음은 무엇인가?"

## 3. Tone Style
- 단정하지만 따뜻하게
- 불안 유발 금지: "큰일 난다" 류 표현 금지
- 실행 중심: "하면 좋다"보다 "이렇게 하자" 선호

## 4. Forbidden Exposure
- NLP/사주/기문/귀문 등 전문용어를 사용자 문장에 직접 노출 금지
