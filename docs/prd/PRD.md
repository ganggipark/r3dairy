# PRD - 개인 맞춤 운세 다이어리 (R³ System)

## 1. 목적
사용자의 출생 정보 기반 흐름(내부 계산)과 사용자 기록(행동/감정/성과)을 결합하여,
매일 "읽고 → 실행하고 → 기록"하는 개인 맞춤 다이어리를 제공한다.
출력 채널은 웹앱 + 인쇄(PDF/제본) 2가지를 동시에 지원한다.

## 2. 핵심 차별점
- (1) 운세(흐름) 제공이 아니라 "하루 운영"을 돕는 실행형 다이어리
- (2) 역할(직업/특성) 기반 가변 콘텐츠: 학생/직장인/프리랜서 등에서 표현/추천이 달라짐
- (3) 인쇄본과 웹앱이 동일한 데이터 구조를 공유 (One Content, Multi Output)
- (4) R³ System 기반: Rhythm → Response → Recode 순환으로 자기조절을 유도

## 3. R³ System 정의
- Rhythm: 오늘의 리듬(흐름)을 설명 가능한 언어로 요약/해설
- Response: 오늘 리듬에서 흔히 발생하는 자동 반응(불안, 충동, 미루기 등)을 인지
- Recode: 언어/질문/기록으로 반응을 재설계(습관화)

### R³ 기본 흐름
Input(출생/설정/기록) →
Rhythm Analysis Engine(내부) →
Content Assembly Engine →
Role Translation Layer →
Output(Web/PDF)

## 4. 사용자 입력(프로필)
### 4.1 필수
- 이름
- 생년월일시
- 성별
- 출생지(도시 단위)
- 역할/직업 (복수 선택 가능)

### 4.2 선택
- 관심 영역 (최대 2개): 일/돈/관계/건강/마음/정리/학습
- 생활 리듬: 아침형/저녁형/불규칙
- 기록 스타일: 짧게/자세히

## 5. 콘텐츠 구성(연/월/일)
### 5.1 연간(Year)
- 연간 흐름 요약(키워드, 조정 포인트)
- 월별 흐름(기회/주의) 시각 요소
- 연간 루틴 가이드(정리/회복/실행 주기)

### 5.2 월간(Month)
- 이번 달 테마 2~4문장 해설
- 월간 우선순위 3개
- 달력 날짜별 키워드/아이콘 요약(앱은 탭 이동)

### 5.3 일간(Day) - 핵심
**일간 페이지는 좌측(오늘의 안내)과 우측(사용자 기록)로 분리한다.**

#### [매우 중요] 좌측 콘텐츠 풍성함 기준
- 좌측(오늘의 안내)은 "읽는 것만으로 하루 방향이 정리되는 수준"을 목표로 한다.
- 좌측은 최소 8개 이상의 독립 블록으로 구성한다.
- 좌측 본문 텍스트는 최소 400~600자(권장 700~1200자) 범위를 목표로 한다.
- 카드형 한 줄 요약만 나열하는 방식 금지. 설명형 문단을 포함해야 한다.

#### 우측(사용자 기록) 기준
- 사용자가 직접 쓰는 공간을 우선 확보한다.
- 최소한의 가이드(질문 1~2개)만 제공한다.

## 6. 역할(직업/특성) 기반 가변 콘텐츠
- "리듬(흐름)" 자체는 동일하지만, 표현/추천/예시/질문이 역할에 맞게 변형된다.
- 학생: 학습/집중/페이스 관리 강조, 계약/리더십/보고 표현 최소화
- 직장인: 업무/관계/결정/보고 강조, 시험/과제 표현 제거
- 프리랜서/자영업: 결정/계약/창작/체력 강조
- 공통: 건강/관계/정리/감정 조절은 모든 역할에 존재

## 7. 법적/표현 정책
- 사용자 노출 텍스트에서 NLP/사주명리/기문둔갑 등의 전문 용어를 사용하지 않는다.
- 내부 계산 모듈에서만 사용 가능(문서화는 legal/TERMINOLOGY_POLICY.md 준수)

## 8. 알림(웹앱)
- 일간 알림(사용자 설정 시간)
- 월간/주간 요약 알림(옵션)
- 알림 문구는 역할별 변형 적용

## 9. MVP 범위
- 사용자 프로필 입력
- 오늘/이번달/올해 화면
- 일간(좌/우) 페이지 렌더링
- 역할 기반 문장 변형
- PDF 출력(1차: 월간+일간)

## 10. 확장
- 365일 전량 생성/저장
- 종교/단체 템플릿 모듈
- 기록 기반 리포트(월간 회고/패턴)
- 프리미엄 인쇄 에디션
